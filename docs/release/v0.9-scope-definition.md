# v0.9 Scope Definition

## Status

**Planning** — not yet a release.

`v0.8.0-review-preview` が最新の review-preview タグである。
本文書は v0.9 の作業範囲を定義するものであり、v0.9 の実装完了を宣言するものではない。

## 目的

v0.9 では、v0.1〜v0.8 で構築した CAH 参照実装の文書・責務説明・未解決領域の可視化を進める。
新たな実行機能の追加は最小限とし、既存構造の「読める化」を優先する。

## v0.8.0-review-preview から引き継ぐ前提

- ToolProxy は副作用の単一隘路である
- PolicyGate / RuntimeBoundary / ServiceRuntime / ExecutionBackend の責務分離は維持する
- Koguchi は security sandbox ではない
- Dashboard / HTML Report は control plane ではない
- 186 tests, 25 source files, 28 ADRs は全通過している

## v0.9 候補項目

### 優先 (Candidate)

| 項目 | 内容 |
|------|------|
| Documentation Polish | README review flow 整理、architecture 図の更新、ADR 読み順の再確認 |
| 責務説明強化 | ToolProxy / RuntimeBoundary / ServiceRuntime / ExecutionBackend の境界を docs で明示 |
| "not sandbox / not control plane" 継続明示 | 全主要文書で制約を確認・補強 |
| Example index | 全 example の一覧と実行方法を文書化 |

### 任意 (Optional)

| 項目 | 内容 |
|------|------|
| Backend-specific Reconciliation | filesystem diff reconciliation の設計（実装は v0.10 以降） |
| Rust Chokepoint Protocol Stabilization | schema versioning, error handling 標準化 |
| Dashboard documentation expansion | `docs/dashboard.md` の充実 |
| Example refinement | example 間の一貫性向上 |

### 延期 (Deferred / post-v0.9)

| 項目 | 理由 |
|------|------|
| Full sandbox implementation | OS-level isolation は Rust/seccomp/container に依存 |
| Full control plane implementation | dashboard は observation plane に留める |
| Production-grade backend orchestration | 複数 backend の動的切り替えは未設計 |
| Multi-backend automatic reconciliation | backend-specific diff の設計が先 |
| Web dashboard server | static HTML report で十分 |
| Remote API server | local-first を維持 |
| Cryptographic audit sealing | 改ざん検出は v1.0 以降 |
| Compliance certification | CAH は compliance tool ではない |

## stable release へ進むための条件

v0.9 完了後に v1.0 へ進むには、以下が必要：

- 全主要コンポーネントの責務分離が docs で明示されている
- "not sandbox / not control plane" が全 entry point で明記されている
- ADR 読み順が整備され、初見の開発者が設計意図を追える
- 残 TODO が deferred / post-v1.0 に分類されている
- backward-incompatible な変更がない

## RDE 観点での注意点

- v0.9 は「完成」ではなく「読める化」のフェーズである
- sandbox / control plane の未実装を隠蔽しない
- 文書強化によって、あたかも全機能が実装済みであるかの印象を与えない
- review-preview と stable release の区別を継続する
