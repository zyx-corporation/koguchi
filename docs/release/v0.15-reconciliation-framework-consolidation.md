# v0.15 Reconciliation Framework Consolidation

## Status

**Consolidation** — no new runtime behavior.

v0.15 consolidates the reconciliation framework introduced across v0.10–v0.14. It does not add new runtime behavior. It documents the current read-only, review-focused, non-production framework boundary.

## Current Full Stack

```
Scheduler → ToolProxy → PolicyGate → ExecutionBackend
  → Reconciliation → AuditStore/ResultStore → CLI → Human Reviewer
```

- Scheduler: timing/request origin. ToolProxy: chokepoint. PolicyGate: observation boundary.
- ExecutionBackend: read-only observation. Reconciliation: comparison.
- AuditStore/ResultStore: observation artifact storage. CLI: read-only inspection.
- Human Reviewer: decision layer.

## Implemented Spike Surfaces

| Module | Scope |
|--------|-------|
| `reconciliation/filesystem_diff.py` | expected vs observed comparison, DiffKind, ReconciliationStatus, no auto-repair |
| `toolproxy/reconciliation.py` | ToolProxy-facing request handler, kind/mode validation, boundary flags, no enforcement |
| `scheduler/reconciliation.py` | Scheduler-originated path, routes via ToolProxy, all actions false |
| `persistence/reconciliation_records.py` | in-memory stores, boundary flags, duplicate/unknown id behavior |
| `cli/reconciliation.py` | list/show/audit, mismatch exits 0, prohibited command rejection |

## Non-production Boundary

The reconciliation framework is not production-ready. It does not provide production backend reconciliation, automatic repair, retry policy, durable persistence, stable CLI, remote API, Rust protocol, crypto sealing, control plane, or security sandbox.

## Release Claims

**Allowed:** Koguchi has a design-gated read-only reconciliation framework scaffold with tested spikes.

**Prohibited:** Koguchi has production reconciliation / automatic repair / retry / correctness decision / control plane / security sandbox.

## Remaining Work Before Production
