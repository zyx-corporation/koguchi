# v0.5.0-observation-preview

Koguchi v0.5.0-observation-preview is an Observation Preview of the Context-Aware Harness reference implementation.

It provides a read-only operational awareness layer over audit records, reconciliation jobs, and Rust chokepoint availability.

## Highlights

- ToolProxy as the single side-effect chokepoint
- PolicyGate / RuntimeBoundary / ServiceRuntime separation
- Persistent Audit Store
- Reconciliation Scheduler
- Rust Chokepoint Spike
- Dashboard Observation Plane
- JSON serializable dashboard snapshot
- Text dashboard report
- ADR-001 to ADR-025

## Try it

```bash
ruff check .
pytest
mypy src

python examples/minimal_tool_proxy.py
python examples/persistent_audit_store.py
python examples/reconciliation_scheduler.py
python examples/rust_chokepoint_spike.py
python examples/dashboard_observation.py

cargo test --manifest-path crates/koguchi-chokepoint/Cargo.toml
cargo build --manifest-path crates/koguchi-chokepoint/Cargo.toml
```

## Dashboard scope

The dashboard is read-only.

It does not:

- execute tools
- approve actions
- mutate audit logs
- rerun reconciliation jobs
- repair or rollback state
- call the Rust chokepoint
- provide a web server
- provide a remote API

## Security and limitations

Koguchi is not a security sandbox.

Python RuntimeBoundary is best-effort.
Rust Chokepoint is an optional spike, not a production sandbox.
Dashboard Observation Plane is read-only and local-first.

See:

- `docs/known-limitations.md`
