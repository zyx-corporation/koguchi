# v0.17 Stable Schema / Protocol Design Gate

## Status

**Design Gate** — not an implementation. No stable protocol is introduced.

v0.17 defines the stable schema/protocol design gate for Koguchi's read-only reconciliation framework. It defines which records may become schema-stabilized, how versioning should work, and what compatibility guarantees must be reviewed before durable persistence or stable CLI work proceeds.

## Candidate Stable

**Enums:** `DiffKind`, `ReconciliationStatus`

**Records:** `ReconciliationAuditRecord`, `ReconciliationResultRecord`, `FilesystemDifference`

**Not yet stable:** ToolProxy/Scheduler/CLI surfaces (integration policy not yet finalized).

## Versioning Policy (Draft)

- `schema_version` field required on top-level records (format: `koguchi.reconciliation.v0`)
- Incompatible change → version bump
- Additive optional fields allowed within same major
- Enum value removal or meaning change is **prohibited**
- Unknown optional fields ignored by consumers
- Required field addition is breaking

## Backward Compatibility Policy

**Backward-compatible:** optional field addition, doc clarification, new boundary flag.

**Breaking:** required field removal/rename, enum removal, enum meaning change, status interpretation change, boundary flag polarity change.

**Invariant:** `matched` ≠ approval, `mismatch_observed` ≠ failure, `review_focus_only` ≠ control action.

## Migration Policy (Draft)

Migration must preserve observation semantics, not convert mismatch → failure or matched → approval. No retry/repair/deploy decisions. Explicit and reviewable.

## Protocol Boundary

Not a Rust protocol, remote API, wire protocol, or production persistence. Defines only schema stabilization requirements.

## Serialization Policy (Draft)

External representation: JSON-compatible. YAML examples remain docs-only. Python dataclasses are implementation scaffolds, not stable external protocol.

## Boundary-preserving Fields

Audit record: `boundary: {read_only: true, no_repair: true, no_retry: true, no_control_decision: true}`

Result record: `boundary: {review_focus_only: true, correctness_decision: false, enforcement: false, approval: false, rejection: false}`
