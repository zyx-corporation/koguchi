## Boundary

Read-only. Not production CLI. No store mutation, retry, repair, approve, reject, deploy, commit, rollback, delete, correctness decision, control plane, or security sandbox.

## Exit Code

- `0`: success (regardless of reconciliation status)
- `1`: usage error or record not found
- `2`: internal error

`mismatch_observed` → 0. Reconciliation status is not process failure.

## Display Interpretation

Matched ≠ approval. Mismatch ≠ failure. Missing ≠ repair instruction. Audit gap ≠ security verdict. Inconclusive ≠ correctness decision.
